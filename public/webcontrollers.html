Web controllers

Weber associates each URL with a action of a Web controller. Where controller - is a just Elixir module, and action is function from this module.

For example we have following routing set up: If Weber got request for "/" it calls `action` function from `Simpletodo.Main` controller module. Every action passes 2 arguments:

  • Http method
  • Bindings list

And must return tupple with 2 elements:

  • {:render, [project: "simpleTodo"], [{"HeanderName", "HeaderVal"}]} - Render views with the same name as controller and sends it to response.
  • {:json, [response: "ok"], [{"HeanderName", "HeaderVal"}]} - Weber convert keyword to json and sends it to response.
  • {:redirect, "/new_location"} - Weber sends redirect.
  • {:render_inline, "foo <%= bar %>", [bar: "baz"]}, []} - Render inline template.
  • {:file, path, headers} - Send file in response.
  • {:text, data, headers} - Sends plain text.
  • {:nothing, ["Cache-Control", "no-cache"]} - Sends empty response with status 200 and headers.
